### §1. Purpose and Scope

This document defines every data structure in the antiphoria slop-research-factory
as implementation-ready Python type definitions. A coding assistant receiving this
document should be able to produce syntactically valid, type-checked Python code
for the full state layer without consulting any other source.

**What this document decides:**

- The exact fields, types, optionality, and serialization format of every structured
  payload that crosses a component boundary.
- Which types are Pydantic `BaseModel` subclasses (validated external/LLM inputs)
  versus plain dataclasses or TypedDicts (internal state).
- The JSON-on-disk representation for every workspace artifact.
- Enum definitions for all constrained string fields.

**What this document does NOT decide:**

- Prompt text (D-3).
- Which verification tasks map to which confidence tiers (D-4).
- The exact CLI interface of the provenance engine (D-5).
- Deployment configuration (D-9).

### §2. Design Principles

| Principle | Rationale | Enforcement |
|---|---|---|
| **JSON-serializable everywhere** | Every type must be writable to disk and readable from disk without custom serializers. The workspace is the source of truth across crash-recovery boundaries. | All fields use `str`, `int`, `float`, `bool`, `list`, `dict`, `None`, or `enum.value`. No raw `bytes`, `Path`, `datetime` objects in serialized form. Paths serialize as POSIX strings. Timestamps serialize as ISO 8601 strings. |
| **Pydantic for LLM outputs** | The Verifier's structured output must be coerced via Instructor. Pydantic provides runtime validation, JSON Schema generation (for Instructor's function-calling mode), and clear error messages when the LLM produces malformed responses. | All types that an LLM is expected to produce are `pydantic.BaseModel` subclasses. |
| **Dataclasses for internal state** | Internal orchestration state (FactoryState, SealedStepReceipt) does not need Pydantic's validation overhead. Plain `@dataclass` with `asdict()` serialization is sufficient and faster. | All types that only the factory code produces and consumes are `@dataclass(frozen=True)` or `@dataclass`. |
| **Append-only semantics for message history** | Per D-0 §3, the `messages` field is append-only. No message may be deleted or modified after insertion. | The `messages` field is a `list` that is only ever extended via `+` or `.append()`. The orchestrator never slices or reassigns elements. |
| **Graduated confidence, never binary truth** | Per D-1 §6 (Non-Defense 4) and D-1 §10 (requirement for D-2), the Verifier must not produce a single `is_hallucinated: bool`. It must produce graduated confidence per dimension. | The `VerifierOutput` model uses `float` confidence fields and a three-valued `Verdict` enum, not booleans. |
| **Configuration overrides are first-class data** | Per D-1 §10, any deviation from default settings must be recorded in the state and propagated to the HAI Card and manifest. | `FactoryConfig` is a frozen dataclass. `FactoryState` carries a `config` field. The finalization node diffs the config against defaults and writes `configuration_overrides` into the manifest. |

### §3. Enumerations

All enums serialize to their string `.value` in JSON.

#### 3.1 `Verdict`

Directly modeled on Aletheia's Verification-and-Extraction prompt
(Feng et al., 2026a, Appendix A). Three values, no more.
Enum: Verdict
CORRECT  — "The solution is flawless, completely rigorous, and
requires no changes."
FIXABLE  — "The core approach is sound, but it contains minor
errors, skips necessary steps, or lacks formal
academic rigor."
WRONG    — "The solution is fundamentally flawed, relies on
invalid logic, or cannot be salvaged without a
complete rewrite of the core approach."
**Routing semantics (defined in D-0 §4):**

| Verdict | Next node | State mutation |
|---|---|---|
| `CORRECT` | `finalize_manifest` | — |
| `FIXABLE` | `reviser_node` in `targeted_repair` mode | `revision_count += 1` |
| `WRONG` | `reviser_node` in `full_rewrite` mode OR `human_rescue_queue` | `rejection_count += 1`; if `rejection_count >= config.max_rejections` → rescue |

This aligns with D-0's updated blueprint semantics: `WRONG` does
NOT mean "skip the Reviser entirely." It means the next rewrite
must discard the prior draft and regenerate from the brief plus
critique, rather than patching the existing draft.

#### 3.2 `StepType`

Per D-1 §10 requirement: auditors must distinguish node types structurally.
Enum: StepType
GENESIS          — The root of the hash chain, created at run
initialization.
PRE_GENERATOR    — Intent seal before Generator inference.
POST_GENERATOR   — Outcome seal after Generator inference
(includes think trace).
PRE_VERIFIER     — Intent seal before Verifier inference.
POST_VERIFIER    — Outcome seal after Verifier inference.
PRE_REVISER      — Intent seal before Reviser inference.
POST_REVISER     — Outcome seal after Reviser inference
(includes think trace).
TOOL_CALL        — Seal of an external tool invocation
(e.g., Crossref lookup, Tavily search).
HUMAN_GATE       — Seal of a human review decision.
MANIFEST         — The final compiled manifest seal.

#### 3.3 `RunStatus`

Enum: RunStatus
INITIALIZING     — Run created, genesis sealed, awaiting first
Generator invocation.
GENERATING       — Generator node is executing.
VERIFYING        — Verifier node is executing.
REVISING         — Reviser node is executing.
AWAITING_HUMAN   — Run paused in human rescue queue.
FINALIZING       — Manifest compilation in progress.
COMPLETED        — All artifacts written, manifest sealed.
FAILED           — Unrecoverable error. Chain may be partial.
NO_OUTPUT        — The system declared inability to produce
output (Aletheia-style self-filtering).
The `NO_OUTPUT` status is a first-class terminal state, not an error.
Per Feng et al. (2026b, §2.2): "Aletheia often admits failure to solve
a problem, a key feature that improved the efficiency of our human-AI
research collaborations." The factory must be able to declare: "I cannot
produce a meaningful draft for this brief."

**Legal transition table:**

| From \ To | `INITIALIZING` | `GENERATING` | `VERIFYING` | `REVISING` | `AWAITING_HUMAN` | `FINALIZING` | `COMPLETED` | `FAILED` | `NO_OUTPUT` |
|---|---|---|---|---|---|---|---|---|---|
| `INITIALIZING` | — | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ | ✅ | ✅ |
| `GENERATING` | ❌ | ❌ | ✅ | ❌ | ❌ | ❌ | ❌ | ✅ | ✅ |
| `VERIFYING` | ❌ | ❌ | ❌ | ✅ | ✅ | ✅ | ❌ | ✅ | ❌ |
| `REVISING` | ❌ | ❌ | ✅ | ❌ | ❌ | ❌ | ❌ | ✅ | ✅ |
| `AWAITING_HUMAN` | ❌ | ✅ | ❌ | ✅ | ❌ | ✅ | ❌ | ❌ | ✅ |
| `FINALIZING` | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | ✅ | ✅ | ❌ |
| `COMPLETED` | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | — | ❌ | ❌ |
| `FAILED` | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | — | ❌ |
| `NO_OUTPUT` | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | — |

Any transition not listed above is illegal and MUST raise an
orchestrator error rather than mutating the state silently.

#### 3.4 `CitationCheckResult`

Enum: CitationCheckResult
VERIFIED           — DOI resolves and metadata matches the claim.
METADATA_MISMATCH  — DOI resolves but author/title/year do not
match what was cited.
NOT_FOUND          — DOI does not resolve; no matching record in
Crossref or Semantic Scholar.
CLAIM_UNSUPPORTED  — Paper exists, but the cited claim does not
appear to be supported by the abstract
(LLM judgment; low-confidence).
INCONCLUSIVE       — Unable to determine (API timeout, ambiguous
metadata, etc.).

#### 3.5 `ConfidenceTier`

A human-readable bucketing of the float confidence score, for use
in the HAI Card and provenance report.
Enum: ConfidenceTier
HIGH        — confidence >= 0.8
MEDIUM      — 0.5 <= confidence < 0.8
LOW         — 0.2 <= confidence < 0.5
VERY_LOW    — confidence < 0.2

Boundary behavior is explicit: `confidence == 1.0` maps to
`HIGH`, and `confidence == 0.0` maps to `VERY_LOW`

### §4. Configuration Schema

The configuration is a frozen dataclass. Once a run begins, the
configuration is sealed into the genesis step and cannot be modified.
Enum: CheckpointBackend
SQLITE   — JSON + local checkpoint files (Phase 1 default)
POSTGRES — Database-backed checkpoints (Phase 2)

@dataclass(frozen=True)
FactoryConfig:

# --- Model topology ---

generator_model:        str   = "deepseek/deepseek-r1"
verifier_model:         str   = "google/gemini-2.5-flash"
reviser_model:          str   = "deepseek/deepseek-r1"

# Note: reviser_model defaults to same as generator_model

# (per D-0 §4, the Reviser uses the same model as the

# Generator with a different system prompt)

# --- Loop limits ---

max_rejections:         int   = 3

# WRONG verdicts before human rescue

max_revisions:          int   = 5

# FIXABLE verdicts before human rescue

# (separate counter from rejections)

max_total_cycles:       int   = 10

# Absolute loop cap regardless of verdict type

max_total_tokens:       int | None   = None

# Optional hard cap across all LLM calls. If exceeded

# route to human rescue or terminate as configured

max_total_cost_usd:     float | None = None

# Optional cost budget across the entire run

# --- Verifier behavior ---

verifier_confidence_threshold: float = 0.8

# Below this, a CORRECT verdict from the LLM is

# demoted to FIXABLE by the routing logic

enable_citation_checking:      bool  = True

# Whether the Verifier performs external citation lookups

citation_check_sources:  tuple[str, ...] = (
    "crossref", "semantic_scholar"
)

# Per D-1 §10 / Attack 3: multi-source verification

# is mandatory. Single-source is insufficient

enable_tavily_search:          bool  = True

# Whether the Verifier uses Tavily for general

# fact-checking beyond citation metadata

weight_logical_soundness:      float = 0.35
weight_mathematical_rigor:     float = 0.25
weight_citation_accuracy:      float = 0.20
weight_scope_compliance:       float = 0.15
weight_novelty_plausibility:   float = 0.05

# These weights parameterize D-4's confidence composition

# They are first-class config so overrides appear in the

# manifest and HAI Card

# --- Output control ---

target_length_words:    int   = 5000
capture_think_tokens:   bool  = True

# Whether to capture and seal <think> traces

# --- Provenance ---

enable_provenance:      bool  = True

# Can be disabled for development/testing ONLY

# Per D-1 §10 / D-9: disabling requires the env var

# ANTIPHORIA_I_UNDERSTAND_NO_PROVENANCE=true

hash_algorithm:         str   = "sha256"

# --- Infrastructure ---

workspace_base_path:    str   = "./workspaces"
checkpoint_backend:     CheckpointBackend = CheckpointBackend.SQLITE

# Enum, not a free-form string, to prevent typo-driven

# configuration drift

**Default detection for override reporting:**

The finalization node computes `configuration_overrides` by comparing
the run's `FactoryConfig` against `FactoryConfig()` (all defaults).
Any field where the run value differs from the default value is
recorded as an override.

If an operator explicitly supplies a value equal to the default, it is
NOT recorded as an override. Only effective deviations are reported.

**Loop-limit precedence:**

1. `max_rejections`
2. `max_revisions`
3. `max_total_tokens` / `max_total_cost_usd` (if configured)
4. `max_total_cycles`

If multiple limits are breached simultaneously, the earliest item in
the precedence list determines the recorded rescue reason.

### §5. Input Schema: `ResearchBrief`

The research brief is the sole human input that initiates a run.
It is a Pydantic model because it arrives from an external source
(CLI argument, web form, Redis queue message) and must be validated.
class ResearchBrief(BaseModel):
"""The human-authored research brief that initiates a factory run.
This is the ONLY human intellectual contribution to the generation
process. Everything downstream is autonomous. The brief is sealed
as the first post-genesis artifact in the chain.
"""

# --- Required fields ---

thesis:           str

# The central claim, question, or hypothesis to investigate

# Free-form text. No length limit, but the Generator will

# receive this verbatim as part of its system prompt

# --- Optional fields ---

title_suggestion:  str | None          = None

# A suggested title. The Generator may override this

outline:           list[str] | None    = None

# An ordered list of section headings or structural

# directives. If None, the Generator determines structure

key_references:    list[str] | None    = None

# A list of DOIs, arXiv IDs, or full citation strings

# that the human considers relevant. The Verifier will

# check these alongside any citations the Generator adds

constraints:       str | None          = None

# Free-form text describing scope limits, methodological

# requirements, or stylistic directives

# Example: "Focus on results from 2020 onward. Use only

# elementary techniques. Target audience: advanced

# undergraduates."

target_venue:      str | None          = None

# The intended publication venue, if any

# Example: "JAIGP", "arXiv preprint", "internal report"

# This affects the Generator's formatting and the Verifier's

# citation standards

domain:            str | None          = None

# The mathematical or scientific domain

# Example: "algebraic topology", "statistical mechanics"

# "combinatorial number theory"

# Used to calibrate the Verifier's confidence thresholds

# and tool selection

class Config:
    json_schema_extra = {
        "examples": [
            {
                "thesis": "Investigate whether the pretzel knot "
                          "P(-3, 5, 13) has infinite order in "
                          "the smooth concordance group.",
                "key_references": ["arXiv:2301.12345" ],
                "domain": "low-dimensional topology",
                "constraints": "Use only classical invariants. "
                               "Do not invoke Heegaard Floer "
                               "homology.",
                "target_venue": "JAIGP"
            }
        ]
    }
**Validation rules:**

- `thesis` must be non-empty after stripping whitespace.
- `thesis` must be at most 10_000 characters after stripping.
- `key_references`, if provided, must each be non-empty strings.
- `target_length_words` in config must be >= 100.

**Reference validator sketch:**

```python
@field_validator("thesis")
@classmethod
def thesis_not_empty(cls, v: str) -> str:
    v = v.strip()
    if not v:
        raise ValueError("thesis must be non-empty")
    if len(v) > 10_000:
        raise ValueError("thesis exceeds 10_000 character limit")
    return v
```

### §6. Core Orchestration State: `FactoryState`

The central state object that flows through the graph. Every node
reads from it and returns a mutated copy. It is a dataclass, not
a Pydantic model, because it is never produced by an LLM.

Recommended helper type for append-only collections:

```python
class AppendOnlyList(list):
    """List wrapper that forbids overwrite, delete, and slice assignment."""

    def __setitem__(self, key, value):
        raise TypeError("AppendOnlyList does not support item reassignment")

    def __delitem__(self, key):
        raise TypeError("AppendOnlyList does not support deletion")

    def insert(self, index, value):
        if index != len(self):
            raise TypeError("AppendOnlyList only permits append-at-end semantics")
        super().insert(index, value)
```

@dataclass
FactoryState:

# --- Identity ---

run_id:              str

# UUID4, generated at run initialization

# Immutable after creation

status:              RunStatus

# Current lifecycle phase. Updated by the orchestrator

# at each node entry and exit

# --- Configuration (immutable after genesis) ---

config:              FactoryConfig

# Frozen. Sealed into the genesis step

# --- Brief (immutable after genesis) ---

brief:               dict

# The ResearchBrief serialized via .model_dump()

# Stored as dict for JSON compatibility

# --- Provenance chain ---

step_index:          int

# Monotonically increasing counter. Incremented by +1

# at every seal event (pre-seal, post-seal, tool-call)

# The genesis seal is step_index=0

latest_hash:         str

# The cryptographic hash output of the most recent seal

# Updated after every seal event

# Initialized to "" before the genesis seal

# --- Loop counters ---

cycle_count:         int            = 0

# Total number of Generator→Verifier (or Reviser→Verifier)

# complete cycles executed. Incremented when the Verifier

# returns any verdict

rejection_count:     int            = 0

# Number of WRONG verdicts received. Triggers human rescue

# when >= config.max_rejections

revision_count:      int            = 0

# Number of FIXABLE verdicts received. Triggers human rescue

# when >= config.max_revisions

# --- Content ---

current_draft:       str | None     = None

# The latest Markdown draft text. Updated after Generator

# or Reviser execution

current_think_trace: str | None     = None

# The <think> token sequence from the most recent

# Generator or Reviser call. Stored separately from the

# draft for atomic sealing (per D-1 §6, Non-Defense 7)

current_critique:    dict | None    = None

# The most recent VerifierOutput, serialized as dict

# Injected into the Reviser's system prompt

current_extracted_citations: list[dict] = field(default_factory=list)

# The authoritative citation list for the CURRENT verifier cycle

# Populated by the citation extraction step before tool checks

# Each entry is a serialized CitationEntry

# This list, not the Verifier's structured output, is the

# single source of truth for which citations were checked

# --- Running totals / budget enforcement ---

total_input_tokens:   int            = 0
total_output_tokens:  int            = 0
total_think_tokens:   int            = 0
total_tool_call_seconds: float       = 0.0
total_wall_clock_seconds: float      = 0.0
total_estimated_cost_usd: float      = 0.0

# --- Message history (append-only) ---

messages:            AppendOnlyList[dict] = field(default_factory=AppendOnlyList)

# Every LLM interaction (prompt + response) is appended

# here as a dict with keys

# {

# "role":       "generator" | "verifier" | "reviser"

# "step_index": int

# "timestamp":  str (ISO 8601)

# "model":      str

# "prompt_hash": str

# "response_hash": str

# "token_counts": {

# "input": int

# "output": int

# "think": int | null

# }

# }

# The actual prompt and response text are stored on disk

# in the workspace, not in this list (to avoid state bloat)

# The hashes allow verification that the disk files match

# --- Discovered citations ---

citation_checks:     AppendOnlyList[dict] = field(default_factory=AppendOnlyList)

# Accumulated citation verification results across all

# cycles. Each entry is a CitationCheckEntry (see §8)

# serialized as dict. Append-only

# --- Workspace path ---

workspace:           str            = ""

# Absolute POSIX path to the run's workspace directory

# Set during initialization

# Example: "/workspaces/a1b2c3d4-e5f6-7890-abcd-ef1234567890"

# --- Timestamps ---

created_at:          str            = ""

# ISO 8601 timestamp of run initialization

updated_at:          str            = ""

# ISO 8601 timestamp of most recent state mutation

# --- Error tracking ---

last_error:          str | None     = None

# If the run enters FAILED status, this field contains

# the error message or traceback summary

**State mutation contract:**

The LangGraph-backed orchestrator must enforce:

1. `run_id`, `config`, and `brief` are set at initialization
   and never modified.
2. `step_index` only increases.
3. `latest_hash` is only updated by the seal wrapper function.
4. `messages` is only extended, never truncated or mutated.
5. `citation_checks` is only extended, never truncated or mutated.
6. `updated_at` is refreshed at every state mutation.
7. `current_extracted_citations` is replaced per Verifier cycle
   and is authoritative only for the current cycle.
8. `total_*` counters are increment-only.

**Serialization:**

The state is serialized to JSON via `dataclasses.asdict()` and
written to `{workspace}/state.json` after every node execution.
This file is the crash-recovery checkpoint for Phase 1 (SQLite
checkpointing replaces this in Phase 2).

State writes MUST be atomic:

```python
tmp = workspace / "state.json.tmp"
tmp.write_text(json.dumps(asdict(state), indent=2), encoding="utf-8")
tmp.replace(workspace / "state.json")
```

Crash-recovery deserialization MUST reconstruct nested dataclasses
explicitly:

```python
raw = json.loads(state_json)
config = FactoryConfig(**raw["config"])
state = FactoryState(
    config=config,
    **{k: v for k, v in raw.items() if k != "config"},
)
```

#### 8.3 `CritiqueEntry`

A single specific flaw or observation identified by the Verifier.

Use **closed enumerations** so malformed LLM output fails validation
before routing (JSON Schema `enum` and/or Pydantic `Literal`):

```python
from typing import Literal

CritiqueCategory = Literal[
    "citation_error",
    "logical_gap",
    "mathematical_error",
    "rigor_deficit",
    "formatting",
    "scope_violation",
    "other",
]
CritiqueSeverity = Literal["critical", "major", "minor"]
```

class CritiqueEntry(BaseModel):
"""A single specific observation from the Verifier's critique."""
category:          CritiqueCategory

# "critical" → likely triggers WRONG verdict

# "major"    → likely triggers FIXABLE verdict

# "minor"    → does not affect verdict

severity:          CritiqueSeverity

# Example: "Reference [4] in the introduction"

schema errors, not tolerated strings.

#### 8.4 `VerifierOutput`

The complete structured output of the Verifier node. This is the
most critical Pydantic model in the system. The Verifier LLM is
coerced to produce this exact structure via Instructor.

The structure follows Aletheia's Verification-and-Extraction prompt
(Appendix A of Feng et al., 2026a):

1. **Independent analysis** (in the model's `<think>` trace,
   not in the structured output)
2. **Critique** → `critique_entries` + `critique_summary`
3. **Verdict** → `verdict`
4. **Resolution** → `resolution`
class VerifierOutput(BaseModel):

specific observations before making a judgment call.

**Routing logic interaction with `VerifierOutput`:**
pseudocode:
output = verifier_llm_call(...)  # Returns VerifierOutput
Demotion rule (per D-0 §8.1; threshold branch below)
if output.verdict == Verdict.CORRECT:
if output.verdict_confidence < config.verifier_confidence_threshold:
effective_verdict = Verdict.FIXABLE
else:
effective_verdict = Verdict.CORRECT
else:
effective_verdict = output.verdict
Routing (per D-0 §4C)
match effective_verdict:
case Verdict.CORRECT:
next_node = "finalize_manifest"
case Verdict.FIXABLE:
state.revision_count += 1
if state.revision_count >= config.max_revisions:
next_node = "human_rescue_queue"
elif config.max_total_tokens is not None and state.total_input_tokens + state.total_output_tokens >= config.max_total_tokens:
next_node = "human_rescue_queue"
elif state.cycle_count >= config.max_total_cycles:
next_node = "human_rescue_queue"
else:
next_node = ("reviser_node", "targeted_repair")
case Verdict.WRONG:
state.rejection_count += 1
if state.rejection_count >= config.max_rejections:
next_node = "human_rescue_queue"
elif config.max_total_tokens is not None and state.total_input_tokens + state.total_output_tokens >= config.max_total_tokens:
next_node = "human_rescue_queue"
elif state.cycle_count >= config.max_total_cycles:
next_node = "human_rescue_queue"
else:
next_node = ("reviser_node", "full_rewrite")

There is intentionally NO promotion rule for low-confidence `WRONG`
verdicts. The system fails safe: a Verifier that is unsure about a
fatal flaw may still force a rewrite or human escalation, but it may
not silently upgrade an unsafe draft toward completion

### §15. Versioning and Migration

All on-disk JSON files include a `_schema_version` key at the
top level:

```json
{
  "_schema_version": "0.1",
  "run_id": "a1b2c3d4-...",
  ...
}
```

When the schema changes in a backward-incompatible way, the
version number increments. The crash-recovery loader must check
_schema_version before deserializing. If the version does not
match the running factory's expected version, the loader must:

Attempt automatic migration (if a migration function exists
for that version pair).
If no migration exists, refuse to resume and log an error
explaining the version mismatch.

This prevents silent data corruption when upgrading the factory
software mid-run.

**Required utility contract:**

```python
def serialize_with_version(obj, version="0.1") -> dict:
    data = asdict(obj) if is_dataclass(obj) else obj.model_dump()
    return {"_schema_version": version, **data}

def deserialize_with_version_check(data: dict, expected="0.1") -> dict:
    actual = data.pop("_schema_version", None)
    if actual != expected:
        raise ValueError(f"Schema mismatch: expected {expected}, got {actual}")
    return data
```
